# Frontend I18n

Use this guide when adding or changing user-facing frontend strings.

## Rules

- Use `useAppTranslation()` in React components.
- Use `tDefault(key, defaultText, options?)` outside React.
- Every i18n call must include a default string.
- Do not call raw `t('key')` without `defaultValue`.
- Keep language state in i18next, not Zustand.
- Browser language is detected first; manual language changes persist in local storage.

## Location

```txt
apps/web/src/app/i18n/
  index.ts
  languages.ts
  resources.ts
  resources.generated.ts
apps/web/scripts/
  i18n.config.example.json
  i18n.config.json
  scan-i18n.mjs
  sync-i18n-sheet.mjs
  generate-i18n-resources.mjs
```

## Pattern

```tsx
const { tDefault } = useAppTranslation();

return <h1>{tDefault('todo.overview.title', 'Todos')}</h1>;
```

For non-React code:

```ts
tDefault('common.actions.cancel', 'Cancel');
```

## Key Rules

- Use dot-separated, lower camel case keys: `admin.organizations.createTitle`.
- Choose the key by semantic owner, not by current file path.
- Use `common.*` only for text that is reusable across unrelated domains, such as shared actions, table labels, fields, errors, pagination, or address labels.
- Use `app.*` for app shell, routing, global feedback, loading, and error-boundary text.
- Use domain or surface namespaces for feature/page text, such as `auth.login.*`, `admin.organizations.*`, `admin.metadata.*`, `home.*`, or `notFound.*`.
- Reuse an existing key only when the meaning is the same, not only because the English default text matches.
- Do not create page-level keys for strings owned by shared components, feature components, app commands, or cross-page helpers.
- Keep keys stable when editing default text unless the meaning changed.

## Google Sheet Sync

The scanner depends on the explicit helper shape:

```ts
tDefault('namespace.key', 'Default string');
```

- Do not hide keys in dynamic builders unless the option set is fixed and easy to scan.
- `i18n:scan` must pass before finishing i18n string changes.
- Use scoped scanning when the task is limited to specific files or folders, once supported by the scanner:
  `pnpm --filter web run i18n:scan -- --scope src/pages/login`
- `i18n:generate` writes `resources.generated.ts`; do not hand-edit generated content.
- Generated resources should reflect Sheet translations. Code defaults are fallback text and scanner input, not a replacement for Sheet values.
- Do not change `generate-i18n-resources.mjs` to force code defaults over Sheet translations.
- If generated output contains stale starter text, update or clear the Sheet, then run the sync/generate flow again.
- Manual resources override generated resources.
